实验要求
注册表单功能
用户可以输入用户名、密码、确认密码，点击注册按钮提交数据。
用户点击注册时，通过 AJAX 异步提交用户名、密码和确认密码到服务器进行验证。
前端验证功能
在提交表单之前，检查用户是否填写了用户名、密码和确认密码。
如果没有填写，则提示用户输入相应字段。
校验密码与确认密码是否一致，如果不一致，提示用户。
用户名检查功能
使用 AJAX 向服务器发送请求，异步判断用户名是否已存在。
如果用户名已存在，提示用户。
注册验证功能
使用 AJAX 向服务器发送请求，验证注册信息是否正确。
根据返回结果，显示注册成功或失败的提示消息。
错误提示功能
如果用户输入的用户名已存在，页面应显示错误提示信息，并保持用户输入内容。
如果注册失败，应显示相应的错误提示信息。
实验步骤
1. 创建注册页面结构
目标：设计一个注册表单，包含用户名、密码、确认密码输入框和注册按钮。
步骤：
创建一个包含 username、password、confirm-password 输入框和注册按钮的表单。
使用 div 标签包装表单元素。
2. 表单提交前的前端验证
目标：检查用户是否输入了用户名、密码和确认密码。
步骤：
在表单提交时，检查用户名、密码或确认密码是否为空。
如果存在为空，使用 jQuery 显示提示信息“请输入用户名、密码和确认密码”，阻止表单提交。
校验密码和确认密码是否一致。
如果不一致，提示“密码与确认密码不一致”。
3. 使用 AJAX 进行用户名校验
目标：通过 AJAX 异步判断用户名是否已经存在。
步骤：
设置监听用户名输入框事件，在用户输入用户名后，点击输入框之外时，使用 jQuery 的 $.ajax() 方法向服务器发送 POST 请求，包含 JSON 格式的用户名。
请求参数中的 url: "http://172.31.73.236:5000/user/check_username", contentType: 'application/json'
如果响应成功，response.code === 200 表明用户名已存在，提示“用户名已存在”，否则，隐藏提示信息。
如果响应失败，提示“服务器异常”。
4. 使用 AJAX 提交注册信息
目标：将用户名、密码、确认密码通过 AJAX 提交到服务器进行注册。
步骤：
使用 jQuery 的 $.ajax() 方法向服务器发送 POST 请求，包含 JSON 格式的用户名、密码和确认密码。
请求参数中的 url: "http://172.31.73.236:5000/user/register", contentType: 'application/json'
如果响应成功，response.code === 200 表明注册成功，提示“注册成功”，否则，提示“用户名已存在”。
如果响应失败，提示“服务器异常”。
注意事项
监听事件外部不能有如 $(document).ready(function () {}) 包含。
